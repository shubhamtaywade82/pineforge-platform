# CW10003 — Extract history-dependent call from scope

Source: https://www.tradingview.com/pine-script-docs/errors/CW10003/

**Type:** Compiler warning

**Message:** The function "X" should be called on each calculation for consistency. It is recommended to extract the call from this scope.

## Cause

History-dependent calls inside `if`/`for`/`while`, ternary (`?:`), or `and`/`or` skip bars. Pine commits time-series data only when the call runs. `[]` and internal history then reference **wrong bars**.

Affected: functions using `[]`, `ta.*`, or any internal history.

**Not affected:** pure functions like `math.max()` (no history).

## Bad vs good

```pine
// Bad — ta.sma only runs when close > close[1]
float inconsistentSMA = if close > close[1]
    ta.sma(close, 20)

// Good — call every bar; gate the plot value
float sma = ta.sma(close, 20)
float inconsistentSMA = close > close[1] ? sma : na
```

```pine
// Bad — UDF with source[1] called conditionally
float pastValue = na
if close > open
    pastValue := previousValue(bar_index)

// Good — global call; conditional assignment
float globalPrev = previousValue(bar_index)
float pastValue = na
if close > open
    pastValue := globalPrev
```

Same issue with ternary:

```pine
// Bad
float pastValue = bar_index % 3 == 0 ? previousValue(bar_index) : na
```

## Why not auto-fix?

Forcing every-bar execution would break side effects (`label.new`, `var` mutations, `log.*`) that must run conditionally.

## Codegen rules

- **Always** compute `ta.sma`, `ta.rsi`, `ta.highest`, etc. at global scope
- Use `if` only to **assign** or **plot** results, not to **call** history functions
- If warning persists and behavior is intentional, debug carefully — prefer fixing structure
