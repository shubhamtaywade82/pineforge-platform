# CE10101 — Condition must evaluate to bool

Source: https://www.tradingview.com/pine-script-docs/errors/CE10101/

**Type:** Compilation error

**Message:** The condition of the "X" statement must evaluate to a "bool" value.

## Cause

`if` and `switch` conditions must be `bool`. Pine v6 **removed** implicit numeric-to-bool casting (v5 allowed `0`/`na` → false, nonzero → true).

## Fixes

### Numeric conditions

```pine
// Bad (v6)
newMonth = ta.change(month)
if newMonth
    label.new(bar_index, high, "New month")

// Good — explicit cast
if bool(newMonth)
    label.new(bar_index, high, "New month")

// Good — comparison
if newMonth != 0
    label.new(bar_index, high, "New month")
```

`bool()` rules: `0`, `0.0`, `na` → `false`; any other value → `true`.

### Testing for na

```pine
// Bad
pivot = ta.pivothigh(10, 10)
if pivot
    label.new(bar_index[10], pivot, "Pivot High")

// Good
if not na(pivot)
    label.new(bar_index[10], pivot, "Pivot High")
```

`bool` values are never `na` — use `na()` for undefined checks.

## Codegen rules

- Always write `if close > open`, `if not na(x)`, or `if bool(count)`
- Never `if someFloat` or `if someInt`
- Pivot/signal checks: `if not na(ta.pivothigh(...))`
